Hello! This is a GUI representation of a replication of "Spatial resource heterogeneity creates local hotspots of evolutionary potential" by Emily Dolson and Charles Ofria (2017).
"Hotspot" cells are represented with circles, and regular cells are rectangles.
"Hotspot" refers to places where a particular trait might evolve more easily, due to cpu cycles.
We expected that hotspot cells (the circles) would have higher rates of completed tasks, but that doesn't seem to be the case, though they do accrue points more quickly. Colors indicate which task a cell has successfully solved, black that an organism hasn't solved anything, and white that the cell is empty and without an organism. The exact color-task equivalences are listed at the bottom of this paragraph

The representation above focus on the movement of the hotspot on the grid.
We implemented nine basic tasks to run this program.
The organisms gain 30 points when the NOT task was solved, because it is very rarely solved.
For other tasks (NAND, AND, ORN, OR, ANDN, NOR, XOR, EQU), the organisms gain 15 points

The user can set 5 different features using the configuration panel.
Num Orgs : Determines the number of starting organisms on the grid.
Required Points : Determines the number of points the organisms need to reproduce.
Moving Hotspot Req : Per how many updates does the hotspot move (0 for never)
Random Seed : Determines the random seed of the simulation.
Num Possible Tasks : Determines the number of tasks possible in the simulation.
Some possible things to try include shifting the number of require points upwards/downwards and seeing how visible activity changes and reducing the number of possible tasks to focus on just a few; tasks are added in the order (first to last): AND, EQU, XOR, NOR, ANDN, OR, ORN, NAND, and NOT.
The colors of completed tasks are:

  • red : NOT
  • blue : NAND
  • yellow : AND
  • #009900 : ORN
  • #66ffff : OR
  • #006699 : ANDN
  • #3366ff : NOR
  • #ccffcc : XOR
  • green : EQU
